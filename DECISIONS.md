# 클래스 핏 — DECISIONS

> 형식: `## D-NN (날짜) 무엇을 물었나 → 선택 / 이유 / 영향받는 태스크`

## D-01 (2026-07-19) 프로젝트 루트 위치 → `coding/learnmatch` 저장소 사용
- **선택**: 기존 `intelier/learnmatch` 저장소(learnmatch-zeta.vercel.app 배포 중)를 프로젝트 루트로 사용. (인터뷰 없이 기본값으로 결정)
- **이유**: 동일 서비스(ClassFit)의 프로토타입 2개가 이미 이 저장소에 있고 Vercel 배포도 연결되어 있음. 새 저장소를 만들면 배포·이력이 분산됨.
- **영향**: T-01 (스캐폴드 위치), U-7 (기존 HTML 처리)

## D-02 (2026-07-19) 진단 문항·채점 체계 → 기존 8문항 확장
- **선택**: 프로토타입의 SDT+ZPD 채점 축(자기주도성·수준부담·소진·유능감·사회성·학습스타일·깊이/넓이)을 유지하며 15~20문항으로 확장.
- **이유**: 이미 동작하는 검증된 구조를 재사용해 M1 속도 확보. 근거 문헌 정리는 별도 작업으로 분리(→ U-6 면책 문구와 함께 다룸).
- **영향**: T-02 (문항 정의 — 확장 설계 포함), T-04 (리포트 프롬프트가 이 축을 입력으로 사용)

## D-03 (2026-07-19) 기존 HTML 데모 처리 (U-7) → `legacy/` 폴더 보존
- **선택**: `index.html`(학원 매칭 데모)·`learning_diagnostic_full.html`(진단 데모)을 `legacy/`로 이동해 보존. (T-01 승인 시 기본값으로 확정)
- **이유**: T-02에서 문항·채점 로직을 참조해야 하고, 학원 매칭 데모는 M3/2단계 참고 자산. 단, 저장소를 push하면 Vercel 배포가 Next.js 앱으로 교체됨 — M1 배포(T-08) 전까지 push 보류.
- **영향**: T-02 (참조 원본), T-08 (배포 시 기존 정적 사이트 대체)

## D-04 (2026-07-19) 공유 링크 방식 → 무상태 URL 인코딩 (M1 한정)
- **선택**: 응답 18개를 18자리 숫자 코드로 URL에 인코딩(`/r/021122...`), 열람 시 채점·리포트 재생성. (기본값으로 결정)
- **이유**: M1에는 DB가 없음(Supabase는 T-09). 무상태 방식이면 배포 환경(Vercel serverless)에서도 저장소 없이 동작. 단점: 열람마다 리포트 재생성(LLM 연동 후 비용 발생) → M2에서 `diagnoses.share_token` + 저장된 리포트 방식으로 교체.
- **영향**: T-06 (구현), T-09 (교체 대상), T-14 (OG 이미지도 코드 기반으로 생성 가능), U-4 (공개 범위는 M2 게이팅 때 결정)

## D-05 (2026-07-19) LLM 제공자 → Google Gemini 1순위로 전환
- **선택**: Anthropic 결제 실패(해외결제 문제)로 리포트 생성 LLM을 `gemini-2.5-flash`(Google AI Studio, 무료 티어)로 전환. 폴백 체인: Gemini → Claude(`claude-sonnet-5`, 키·크레딧 확보 시 자동 사용) → mock.
- **이유**: Gemini AI Studio 키는 결제 수단 없이 무료 티어로 발급 가능. Claude 코드는 유지되어 있어 크레딧 확보 시 파일 수정 없이 복귀 가능(1-Pager 원안은 Claude Sonnet 5).
- **영향**: T-05 (완료 기준을 "실제 LLM 리포트"로 해석 — Gemini로 충족 가능), lib/llm.ts (교체 지점)

## D-06 (2026-07-19) Supabase 연결 시점 → T-08 직후로 앞당김
- **선택**: T-09(Supabase 연결)를 M2 결제 작업까지 기다리지 않고 T-08 배포 검증 직후 바로 진행.
- **이유**: 배포 후 실사용자 응답이 어디에도 저장되지 않아 M1 가설 검증 지표(완주 수·응답 분포)를 볼 수 없고, 공유 링크 열람마다 LLM 재호출 비용 발생. 데이터 전략(1-Pager)상 응답·리포트는 핵심 축적 대상.
- **영향**: T-09 (앞당김 — 저장 + 리포트 캐시 + share_token까지 포함), T-06 (무상태 링크 → DB 토큰 링크로 교체 시점도 함께)

## D-07 (2026-07-19) 무료/유료 경계 (U-2) → 공감 후킹까지 무료
- **선택**: 무료 = 헤드라인 + 레이더·축 레벨 + "한눈에 보기" + "이런 모습, 익숙하시죠?" / 잠금(990원) = "어쩌면 의외의 모습" + 축별 읽기 + 조언 + 학원 팁.
- **이유**: "우리 아이 얘기 같다"는 공감을 무료로 증명해 신뢰를 만들고, 가장 강력한 콘텐츠(의외성 인사이트·실천 조언)를 잠가 결제 동기를 극대화. 서술의 맛을 보여줘야 990원 전환이 일어난다.
- **영향**: T-10 (게이팅 구현 경계), T-11 (언락 CTA 위치 — 잠금 시작 지점), U-4 (공유 페이지도 같은 경계 적용 검토)

## D-08 (2026-07-20) 결제 자동 언락 → Groble 웹훅 + 구매 시 질문으로 진단 식별
- **선택**: Groble 웹훅(`payment.completed`)을 받아 자동 언락. 웹훅 페이로드에 커스텀 메타데이터·쿼리파라미터가 없어, 상품의 "구매 시 질문" 기능으로 "진단 결과 링크"를 필수 입력받아 `questionAnswers`에서 share_token을 추출하는 방식 채택.
- **이유**: Groble 지원팀 확인 결과 API 없음, 자동 전달 기능은 전 구매자 동일 링크만 지원(1인 1코드 불가). 웹훅+구매시질문 조합이 유일한 자동화 경로.
- **영향**: T-12 (자동 언락 엔드포인트), 관리자 언락 API(`/api/unlock`)는 웹훅 실패 시 폴백으로 유지
